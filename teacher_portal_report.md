# **3\. تحليل معمق للوحدات الأساسية**

## **3.1 لوحة التحكم الرئيسية — Teacher Dashboard**

|     |     |     |
| --- | --- | --- |
| **لوحة التحكم** | **التقييم** | **8.5 / 10** |

**نقاط القوة:**

- بانر ترحيبي بتدرج لوني احترافي مع اسم المعلم مخصصاً ديناميكياً
- 3 بطاقات إحصائية فورية: إجمالي الطلاب / المجموعات النشطة / الاختبارات
- شبكة 2×2 تجمع أحدث المجموعات مع كود الدعوة وأحدث الاختبارات بادج الحالة
- نظام بانر trial ذكي يحسب الأيام المتبقية لحظة بلحظة ويُخفي نفسه عند التحقق
- Animated components: AnimatedBanner, AnimatedStatCard, AnimatedSection بـ delay متعرج

**جوانب التحسين:**

- إحصائية الاختبارات تعكس totalExams من آخر 5 فقط وليس العدد الإجمالي الحقيقي
- لا يوجد quick action لإضافة طالب مباشرة من الداشبورد
- غياب رسم بياني مبدئي لأداء الطلاب على مستوى الداشبورد

## **3.2 إدارة المجموعات**

|     |     |     |
| --- | --- | --- |
| **المجموعات** | **التقييم** | **9.0 / 10** |

**الميزات الموثقة في الكود:**

- صفحة قائمة المجموعات تستدعي GroupsClient مع حقن البيانات server-side
- صفحة تفاصيل المجموعة تعرض 5 إحصائيات: الطلاب / الاختبارات / المنشور / الحصص / متوسط الأداء

**فجوات مقترحة:**

- لا يوجد خيار نسخ كود الدعوة بنقرة واحدة (Copy to clipboard)

## **3.4 بنك الأسئلة**

|     |     |     |
| --- | --- | --- |
| **بنك الأسئلة** | **التقييم** | **8.5 / 10** |

**نظام الفلترة متعدد الأبعاد:**

- فلتر الصف الدراسي — يُعيد ضبط الوحدة والدرس تلقائياً عند التغيير
- فلتر الفصل الدراسي — ملون بالعنبري للتمييز
- فلتر الوحدة — يظهر فقط عند اختيار صف ومربوط بـ subject_id المعلم
- فلتر الدرس — يظهر فقط عند اختيار وحدة
- فلترة ثانوية: نوع السؤال (5 أنواع) + مستوى الصعوبة + تصنيف بلوم (6 مستويات)
- بانر أخضر يُعلم المعلم بأن العرض مقيّد بمادته فقط
- toggle أسئلتي فقط مع عداد الأسئلة الخاصة
- pagination بـ limit=60 مع زر عرض المزيد

**ملاحظات تقنية:**

- 7 استخدامات لـ : any في الكود — يُوصى باستبدالها بـ interfaces محددة
- دالة buildHref() لبناء URL parameters ذكية تحافظ على الفلاتر الأخرى

## **3.5 شرح الدروس — نظام المحتوى التعليمي**

# **4\. محرك الذكاء الاصطناعي لتوليد الأسئلة**

## **4.1 المعمارية التقنية لـ API التوليد**

الملف: /app/api/teacher/generate-questions/route.ts — أكثر الملفات تعقيداً وأفضلها هندسياً في البوابة.

|     |     |
| --- | --- |
| **الخاصية** | **التفاصيل** |
| نماذج Gemini | gemini-2.5-flash → gemini-flash-latest → gemini-2.0-flash (Fallback تلقائي) |
| مفاتيح API | 4 مفاتيح دوارة عشوائية لتوزيع الحمل وتجنب تجاوز الكوتا |
| مسارات الإدخال | ملف مرفق (PDF/JPG/PNG) أو نص ملصوق من المعلم |
| خيارات التوليد | العدد + أنواع الأسئلة + المستوى المعرفي + تعليمات مخصصة + passage-based |
| صيغة الاستجابة | application/json مع temperature: 0.2 للدقة |
| معالجة الأخطاء | 429/503/403 قابلة للإعادة + 2s delay بين المحاولات |
| فحص الصلاحيات | double-check: JWT + role='teacher' قبل أي معالجة |

# **5\. منظومة الأمان والصلاحيات**

## **5.1 طبقات الحماية في بوابة المعلم**

|     |     |
| --- | --- |
| **الطبقة** | **التطبيق الموثق** |
| Layout Guard | كل صفحة في /teacher/ تبدأ بـ getCurrentProfile() + فحص role==='teacher' |
| Trial Check | layout.tsx يفحص is_verified + subscription_ends_at ويعيد توجيه لـ trial-expired |
| Onboarding Guard | redirect إلى teacher-onboarding إذا لم يُحدد subject_id بعد |
| API Auth | كل route يتحقق من JWT + role قبل المعالجة |
| Data Isolation | جميع استعلامات المجموعات/الاختبارات تُقيّد بـ teacher_id=profile.id |
| Subject Lock | بنك الأسئلة مقيّد دائماً بـ subject_id المعلم — لا يمكن تجاوزه |
| RLS Layer | قاعدة البيانات تُطبق Row Level Security كطبقة أمان أخيرة |

## **5.2 ثغرات أمنية مُحتملة وتوصيات**

|     |     |     |     |
